Exercise 1:

If you load the repos for hapjs say, and then query for another user (your own github account say) - what happens? It looks like it still returns the hapijs repos. In fact, it is appended the second query on to the first. Change this so that we reset the table (emptying its contents) on every request.

HINT: see https://api.jquery.com/empty/

Solution

Insert this at the start of the populateTable function

  $("#repo-table tbody").empty()

Exercise 2:

Rework the table to display the repo name and then the url in a separate column. Make the url a link to the repo web page in github.

A new column in the table:

              <thead>
                <tr>
                  <th>Repo Name</th>
                  <th>Url</th>
                </tr>
              </thead>

A revised populateTable function:

function populateTable(repoList) {
  $("#repo-table tbody").empty();
  for (let i = 0; i < repoList.length; i++) {
    $("#repo-table tbody").append(`
      <tr> 
        <td>  ${repoList[i].name} </td>
        <td>  <a href="${repoList[i].html_url}"> link </a> </td>
      </tr>`);
  }
}

Foursquare

Sign up for a developer account :

  • https://developer.foursquare.com/

Visit:

  • https://foursquare.com/developers/apps

Create an app - you might use the following settings:

As you proceed through creating the app - you will be asked for a credit card. There is no need to enter these details - we will be able to work with an unverified sandbox account.

One created, these keys should be prominently available:

... and take note of them.

Copy/paste this url into a simple text editor (sublime perhaps):

https://api.foursquare.com/v2/venues/explore?near=Waterford,IE&query=&v=20190101

In the editor, modify the url to include ID and SECRET - according to the following pattern:

https://api.foursquare.com/v2/venues/explore?near=Waterford,IE&query=&client_id=YOUR_ID&client_secret=YOUR_SECRET&v=20190101

(replace YOUR_ID and YOUR_SECRET in the above)

Now transfer the url to an browser and you should see results:

We are now making the same request with the credentials correctly inserted.
