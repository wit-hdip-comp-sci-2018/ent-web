DonationService

Core features of the application, like making donations, adding candidates, are best separated out from the application components/custom elements - and centralised into a Service. This can be shared and access from different parts of the application.

Start wit the Candidate management - create a new class in a new folders src/services:

src/services/donation-service.ts

import {Candidate} from "./donation-types";

export class DonationService {
  candidates : Candidate[] = []
}

An instance of this will be created by Aurelia as needed - and a reference to the single instance can be injected into any interested Component/Custom Element as needed.

Here is a revised Candidates Component, which has a DonationService injected by aurelia:

src/views/candidates.ts

import { inject } from 'aurelia-framework';
import { Candidate } from '../services/donation-types';
import { DonationService } from '../services/donation-service';

@inject(DonationService)
export class Candidates {
  candidates: Candidate[];

  constructor(private ds: DonationService) {
    this.candidates = ds.candidates;
  }
}

No other changes are needed - run the app and verify that it works as before.

Accessing DonationService

Now that we have a centralised DonationService, we can access this from other parts of the application. We start with the DonationForm Custom Element:

src/resources/elements/donate-form.ts

import { bindable } from 'aurelia-framework';
import { Candidate, Donation } from '../../services/donation-types';

export class DonateForm {
  amount = '0';
  @bindable
  donations: Donation[] = [];
  @bindable
  paymentMethods: string[];

  @bindable
  candidates: Candidate[];

  selectedMethod = '';
  selectedCandidate : Candidate = null;

  makeDonation() {
    const donation = {
      amount: parseInt(this.amount),
      method: this.selectedMethod,
    };
    this.donations.push(donation);
    console.log(this.donations);
  }
}
