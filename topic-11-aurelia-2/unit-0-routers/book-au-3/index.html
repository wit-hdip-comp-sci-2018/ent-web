Client Side Navigation

We have two components more or less stuck together currently:

That is, they are combined directly in the app.html:

src/app.html

<template>
  <require from="views/candidates"></require>
  <require from="views/donate"></require>

  <div class="ui container">
    <section class="ui raised segment">
      <h3 class="ui headder"> Donation </h3>
    </section>
    <div class="ui basic segment">
      <donate></donate>
      <candidates></candidates>
    </div>
  </div>
</template>

We would like to have these components separated - and a simple menu driven navigation system to allow the user to load which ever is selected:

Each component would be attached depending on the menu bar selection.

Unlike the server rendered application we have been devising to date - we would like all of this to be operating on the client side, without any server round trips.

This is a very common mechanism in SPA frameworks - and they usually provide it via a client side router. This mirrors somewhat the routing you are used to on the server - with paths and what look like action methods - except the entire mechanism is implemented in the browser.

We introduce a router in the next step...

Router

Replace app.ts with the following:

src/app.ts

import { RouterConfiguration, Router } from 'aurelia-router';
import { PLATFORM } from 'aurelia-pal';

export class App {
  router: Router;

  configureRouter(config: RouterConfiguration, router: Router) {
    config.map([
      {
        route: ['', 'donate'],
        name: 'Donate',
        moduleId: PLATFORM.moduleName('views/donate'),
        nav: true,
        title: 'Donate'
      },
      {
        route: 'candidates',
        name: 'candidates',
        moduleId: PLATFORM.moduleName('views/candidates'),
        nav: true,
        title: 'Candidate'
      }
    ]);
    this.router = router;
  }
}

Each entry in the router data structure (the map) includes:

  • the route that will be mapped,
  • the name and title associated with this route
  • the component to be loaded
  • whether the route is to be part of a navigation control

So, in the above

  • '' and 'donate' are mapped to the donation component
  • 'candidates' is mapped to the candidates component

The app View Model must now be reworked to look like this:

src/app.html

<template>
  <require from="views/candidates"></require>
  <require from="views/donate"></require>

  <div class="ui container">
    <section class="ui raised segment">
      <h3 class="ui headder"> Donation </h3>
    </section>
    <div class="ui basic segment">
      <router-view></router-view>
    </div>
  </div>
</template>

We have replaced 

      <donate></donate>
      <candidates></candidates>

with:

     <router-view></router-view>

The router view is an Aurelia component that drives the routing behaviour. The next step will attach a navigation component to the route, and establish the behaviour we are seeking.

Navigation

Define a new Custom Element which encapsulates the Navigation bar:

src/resources/elements/nav-bar.html

<template bindable="router">

  <nav class="ui inverted menu">
    <div class="right menu">
      <div repeat.for="row of router.navigation">
        <a class="${row.isActive ? 'active' : ''} item"  href.bind="row.href">${row.title}</a>
      </div>
    </div>
  </nav>

</template>

This component is attached to the router data structure defined in the app component - and the navigation elements are populated.

Finally - we include this element in the app component:

src/app.html

<template>
  <require from="resources/elements/nav-bar.html"></require>

  <div class="ui container">
    <section class="ui raised segment">
      <h3 class="ui headder"> Donation </h3>
    </section>
    <nav-bar router.bind="router"></nav-bar>
    <div class="ui basic segment">
      <router-view></router-view>
    </div>
  </div>
</template>

This is the new element:

      <router-view></router-view>

The app is now functioning as expected - and we can operate the menu:

Note carefully the structure of the urls in the address bar. Try these directly:

  • http://localhost:8080/#/candidates
  • http://localhost:8080/#/donate
  • http://localhost:8080/#

The # plays a crucial role here - it ensures that the requests are interpreted on the client, not being transmitted to the server.

Open any of the above links in a different browser - you should go directly to the target component.

Donation Type

So far our donations are just numbers:

  donations: number[] = [];

We would like to evolve a more detailed donation, consisting of the donation amount + payment method. This can be appended to the donation-types.ts file:

src/services/donation-types.ts

...

export interface Donation {
  amount: number;
  method: string;
}

We now refactor he Donate component to manage an array of these donations

src/views/donate.ts

import { Donation } from "../services/donation-types";

export class Donate {
  donations: Donation[] = [];
}

The donate-form custom element must not create this type of donation object:

src/resources/elements/donate-form.ts

import { bindable } from 'aurelia-framework';
import {Donation} from "../../services/donation-types";

export class DonateForm {
  amount = '0';
  @bindable
  donations: Donation[] = [];

  makeDonation() {
    const donation = {
      amount: parseInt(this.amount),
      method : 'cash'
    };
    this.donations.push(donation);
    console.log(this.donations);
  }
}

At the moment - all donations have 'cash' as the payment method.

Payment Method

To represent payment methods more generically, we could define an array of possible methods in the donate component:

src/views/donate.ts

import {Donation} from "../services/donation-types";

export class Donate {
  donations: Donation[] = [];
  paymentMethods = ['Cash', 'Paypal'];
}

When including the donate-form custom element we need to bind the new paymentMethods field:

src/views/donate.html

<template>
  <require from="../resources/elements/donations-list"></require>
  <require from="../resources/elements/donate-form"></require>

  <div class="ui stackable two column grid">
    <div class="column">
      <donate-form donations.bind="donations" payment-methods.bind="paymentMethods"></donate-form>
    </div>
    <div class="column">
      <donations-list donations.bind="donations"></donations-list>
    </div>
  </div>
</template>

Now we redo the donate-form custom element accept this new binding:

src/resources/elements/donate-form.ts

import { bindable } from 'aurelia-framework';
import {Donation} from "../../services/donation-types";

export class DonateForm {
  amount = '0';
  @bindable
  donations: Donation[] = [];
  @bindable
  paymentMethods: string[];

  selectedMethod = '';

  makeDonation() {
    const donation = {
      amount: parseInt(this.amount),
      method : this.selectedMethod
    };
    this.donations.push(donation);
    console.log(this.donations);
  }
}

Finally, we populate the radio checkbox in the view model:

src/resources/elements/donate-form.html

<template>
  <form submit.trigger="makeDonation()" class="ui form stacked segment">
    <h3 class='ui dividing header'> Make a Donation </h3>
    <div class="grouped inline fields">
      <div class="field">
        <label>Amount</label> <input type="number" value.bind="amount">
      </div>
    </div>
    <div class="grouped inline fields">
      <div class="field" repeat.for="method of paymentMethods">
        <div class="ui radio checkbox">
          <input type="radio" model.bind="method" checked.bind="selectedMethod">
          <label>${method}</label>
        </div>
      </div>
    </div>
    <button class="ui blue submit button">Donate</button>
  </form>
</template>
